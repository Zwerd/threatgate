Sanity Check - Analyst Behaviour

When an IOC triggers a critical sanity check (e.g. TLD-only, critical infrastructure IP), choose who is blocked and who only sees a warning with the reason.

To protect organization domains and other critical assets from being submitted as IOCs, use Admin → Allowlist. Entries in the Allowlist always get a red block (no override).

Critical anomalies (can block or warn)

  • TLD-only - Blocking entire TLD (.com, .ru, etc.) would break the internet. Block specific domains instead.
  • TLD-only (value) - Value appears to be TLD-only. Block specific domains, not entire TLD.
  • Critical infrastructure IP - e.g. 8.8.8.8, 8.8.4.4 (Google DNS), 1.1.1.1, 1.0.0.1 (Cloudflare DNS). Blocking would break DNS.

Warning-only anomalies (orange alert, submission allowed)

  • Short domain - Very short domain (e.g. co.il, t.co). Possible typo or URL shortener; high risk of blocking legitimate sites. Uses ccSLD-aware logic (co.il, co.uk, com.au, etc. count as one label).
  • Hash length mismatch - Hash length is not 32, 40, or 64 (MD5=32, SHA1=40, SHA256=64). Verify type.
  • Whitespace trimmed - Leading or trailing whitespace was trimmed.
  • URL contains hash - URL contains a SHA-256-, SHA1-, or MD5-like string. Verify this is correct.
  • Bogon/reserved IP - IP in a bogon or reserved range. Not routable on the public internet.
  • Popular/high-traffic domain - e.g. google.com, facebook.com, youtube.com. Blocking could disrupt critical services.
  • Cloud/CDN infrastructure - e.g. amazonaws.com, cloudfront.net, azureedge.net. Blocking may affect many legitimate services.
  • Free email provider - e.g. gmail.com, yahoo.com, outlook.com. Blocking would cut email access for many users.
  • Punycode/IDN domain - Domain uses Punycode (xn--). May be a homograph phishing attack; verify the rendered characters visually.
  • DGA-like domain - Domain label has high entropy or unusual character distribution. Possibly auto-generated by malware.
  • Excessive subdomain depth - 4 or more levels. May indicate DNS tunneling, DGA, or C2 beaconing.
  • URL with embedded credentials - URL contains user:password@host. Security risk; credentials exposed in feed data.
  • URL with raw IP - URL uses a raw IP instead of a domain name. Common in C2 and malware delivery infrastructure.
  • Stale IOC - Permanent IOC active for a long time (e.g. 180+ days). Threat infrastructure rotates; consider reviewing or setting an expiration.
  • Local IP - Private/RFC 1918 address. Blocking might cut internal access. (Feed Pulse)
  • Testnet IP - RFC 5737 documentation range. Verify this is not an example. (Feed Pulse)
  • Defanged - Value contains hxxp, [.], (.), [dot]. Firewall needs normal http and dots. (Feed Pulse)

Allowlist (Admin → Allowlist) always blocks submission for listed entries, regardless of this setting.
