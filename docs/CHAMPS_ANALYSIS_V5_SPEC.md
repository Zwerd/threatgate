# Champs Analysis 5.0 - Operational Hall of Fame 🏛️

## הקונספט

לא דוח אקסל - זה המסך שמוצג על הפלזמה במרכז החדר. משדר: "אנחנו רואים מה אתם עושים, ואנחנו מעריכים מצוינות."

**מטרות:**
- **ניהולית:** תמונת מצב מהירה על תפקוד ה-SOC
- **פסיכולוגית:** הזרקת אדרנלין ואמביציה לצוות, במיוחד בסביבה Offline

## 1. אזור א': המשימה המשותפת (Team HUD) 🤝

**מיקום:** פס רחב בחלק העליון

- **וויזואליה:** בר התקדמות ענק, זוהר ויוקרתי (כמו במשחקי וידאו)
- **תוכן:** יעד שבועי/חודשי
  - **יעד שבועי (מבוסס שבוע קודם):** 100% = כמות שהוכנסה **בשבוע שעבר** (למשל 100 IOCs). השבוע הנוכחי נמדד מול זה - אם הוכנסו 110 → 110%. תמיד מול השבוע שקדם. (שבוע = ISO, שני-ראשון.)
  - **יעד חודשי:** יעד ידני (target_value) + התקדמות ב־30 יום אחרונים.
- **אפקט:** כשמישהו תורם, הבר מתמלא ואפקט נוצץ - "אנחנו ביחד בזה"

**טכני:** טבלה `team_goals` - `id`, `title`, `target_value`, `current_value`, `unit`, `goal_type`, `period` (weekly/monthly). עבור `period=weekly`: ה־API מחשב `target_value` = סך הפעילות בשבוע שעבר, `current_value` = סך הפעילות השבוע (עד היום); אחוז יכול לעבור 100%.

## 2. אזור ב': הלידרבורד הדינמי (The Ladder) 🪜

**מיקום:** סרגל צד שמאל (~25% מהמסך)

- **תצוגה:**
  - תמונת פרופיל (Avatar) בעיגול
  - שם האנליסט
  - ניקוד + מיקום (#N או מדליה)
  - **מדליות (Top 3):** 🥇🥈🥉 בתוך עיגול צבעוני עם אפקט פעימות לב; למקום 1 גם אפקט ברק/זוהר מסתובב
  - **אינדיקטור מגמה:** ליד המיקום - `[ +2 ] ▲` (ירוק) / `[ -1  ] ▼` (אדום), סוגריים מרובעים וזוהר. השוואה ליום האתמול.

**טכני:** טבלה `champ_rank_snapshots` - snapshot יומי של rank/score; מגמה מחושבת מול אתמול.

## 3. אזור ג': כרטיס השחקן (Analyst Spotlight) 🕵️‍♂️

**מיקום:** מרכז המסך (נפתח בלחיצה על אנליסט)

### 3א. טייטל ודרגה (RPG)
- שם + כינוי: "Guy Z. - 🛡️ Network Hunter"
- **כינוי דינמי לפי סוג מזהים:** IP = Network Hunter, Domain = Domain Scout, Hash = Malware Slayer, URL = URL Tracker, YARA = Code Breaker, Email = Phish Hunter
- **מד XP:** "Level 4 → Level 5 (עוד X נקודות)" - 25 רמות (ראו סעיף 5)

### 3ב. גרף השוואה (Overlay)
- גרף קווי: פעילות האנליסט ב־30 הימים האחרונים (צבע ניאון)
- ברקע: גרף אפור עמום = ממוצע צוותי
- כותרת/תווית: **שם האנליסט** vs team avg (לא "You")
- מסר: "מעל הקו" vs "מתחת לקו"

### 3ג. ארון הגביעים (Trophy Cabinet) 🏆

**מיקום:** פאנל צד ימין (רוחב כמו הלידרבורד, 300px). התגים **לא** מופיעים על כרטיס ה-Spotlight - רק בארון הגביעים.

- כל תג: עיגול גדול עם סמל, מסגרת וזוהר בצבע התג, מתחתיו שם התג באותיות לבנות מודגשות
- ריחוף (hover): הסבר קצר באנגלית (מה התנאי להישג)
- תגים מופיעים רק כאשר מתקיים תנאי הפעילות. מעל 20 תגים.

**סטריק וזמן:**
- 🔥 **On Fire:** רצף 5 ימים של הזנת נתונים
- 🌡️ **Warm Streak:** רצף 3-4 ימים
- 🦉 **Night Owl:** פעילות בין 22:00-04:00
- 🌞 **Early Bird:** פעילות בשעות 05:00-07:00
- 🗓️ **Weekend Warrior:** פעילות ביום שישי או שבת

**נפח והתמדה:**
- 💎 **Rare Find:** מזהה "ראשון במערכת" - IP ממדינה (GEO) שעדיין לא הופיעה, או TLD חדש (דומיין/URL), או דומיין אימייל חדש
- 💪 **Dedicated:** 30+ מזהים סה"כ
- 🎖️ **Veteran:** 80+ מזהים סה"כ
- 📅 **Consistent:** פעילות ב־7+ ימים שונים (30 יום אחרונים)
- ⚡ **Ever Present:** פעילות ב־15+ ימים שונים

**ניקוי:**
- ✨ **Clean Slate:** הסרה של לפחות מזהה אחד שפג תוקף
- 🧹 **Janitor:** 5+ הסרות של מזהים שפג תוקף
- 🧼 **Cleanup Crew:** 15+ הסרות

**קמפיינים ו־YARA:**
- 🤝 **Team Player:** לפחות מזהה אחד מקושר לקמפיין
- 🎯 **Campaign Master:** 10+ מזהים מקושרים לקמפיינים
- 📜 **YARA Rookie:** העלאה של לפחות חוק YARA אחד
- 👑 **YARA Master:** 3+ חוקי YARA
- 🏆 **YARA Legend:** 8+ חוקי YARA

**סוגי מזהים (התמחות):**
- 🔐 **Hash Hunter:** 10+ hashes
- 🌐 **Domain Scout:** 15+ דומיינים
- 📍 **IP Tracker:** 25+ כתובות IP
- 🏄 **URL Surfer:** 10+ URLs
- 🎣 **Phish Buster:** 5+ כתובות אימייל

**גיוון:**
- 🎪 **Triple Threat:** הזנה של לפחות 3 סוגי מזהים (IP, Domain, Hash, URL, Email)
- 🌟 **All-Rounder:** הזנה של כל 5 סוגי המזהים

## 4. אזור ד': הטיקר החי (News Ticker) 📣

**מיקום:** פס בתחתית (סגנון BBC Breaking News) - גלילה מימין לשמאל, טקסט מודגש, מפריד ירוק בוהק ` | ` בין הודעות.

**צבעים לפי קטגוריה (מילות מפתח מודגשות):**
- **ירוק (analyst_success):** הצלחות אנליסט - הגשת IOC, YARA, מחיקת פג תוקף, עלייה בדירוג.
- **כתום (important):** הודעות חשובות (להרחבה עתידית).
- **כחול (team):** הישג צוותי - יעד חדש, יעד הגיע ל־25% / 50% / 75% / 80% / 100%.

**סוגי הודעות (דוגמאות):** הגשת IOC, העלאת YARA, מחיקת IOC שפג תוקף, יעד צוותי חדש, שינוי דירוג (עקף X ועלה למקום #N), התקדמות יעד (25%, 50%, 75%, 80%, 100%). תכנון: עד 50 סוגים עם צבעים (מפורט במסמך עבודה).

**לופ:** כל ההודעות ברצף אחד עם מפרידים; תוכן משוכפל לגלילה רציפה (מרקי). טעינה מחדש כל 60 שניות.

כשאין אירועים: placeholder "No recent activity — new submissions, rank changes and goal updates will appear here."

**טכני:** טבלת `activity_events`; `GET /api/champs/ticker?limit=20`. כל הודעה עם `category`: `analyst_success` | `important` | `team`. אבני דרך יעד: 25, 50, 75, 80, 100 (נרשמים פעם ביום לכל milestone).

## 5. לוגיקת ניקוד (Scoring)

סקאלה מוקטנת כדי ששנת פעילות (~10k IOCs) לא תגיע למאות אלפי נקודות.

| סוג | נקודות |
|-----|--------|
| IP/Domain רגיל | 2 |
| IP עם Context/Campaign | 3 |
| YARA Rule | **10-50 לפי איכות** (ראה להלן) |
| מחיקת מזהה פג תוקף | 1 |
| **בונוס רצף:** 3 ימים ברצף | +10% לניקוד היומי |

**YARA לפי איכות:** בהעלאה/עדכון חוק נחשב ציון 10-50 לפי היוריסטיקה: מספר מחרוזות (`$name =`), אורך ומורכבות בלוק `condition` (and/or/not/any/all/of). חוק פשוט → כ־10-18, בינוני → 19-35, מתקדם → 36-50. הערך נשמר ב־`yara_rules.quality_points`. חוקים ישנים (ללא שדה) מקבלים 25.

**רמות (Levels):** 25 רמות. סף מעבר (נקודות מצטברות):
- 1: 0, 2: 200, 3: 450, 4: 750, 5: 1100, 6: 1500, 7: 2000, 8: 2600, 9: 3300, 10: 4100, 11: 5000, 12: 6100, 13: 7400, 14: 9000, 15: 10900, 16: 13100, 17: 15600, 18: 18500, 19: 21800, 20: 25500, 21: 29700, 22: 34400, 23: 39600, 24: 45400, 25: 59000.

מעבר ל־Level 2 דורש ~100 IOCs (200 נקודות). רמה 25 = 59,000 נקודות (יעד ארוך טווח).

**שימור:** Backend Python (`utils/champs.py`) עם שאילתות SQLite (GROUP BY, COUNT).

## 8. Endpoints נדרשים

| Endpoint | תיאור |
|----------|--------|
| `GET /api/champs/leaderboard` | רשימת אנליסטים + rank, score, trend, avatar |
| `GET /api/champs/analyst/<id>` | פרטי אנליסט: XP, level, nickname, activity chart, badges |
| `GET /api/champs/team-goal` | יעד צוותי + התקדמות; **שבועי:** target = שבוע שעבר, current = השבוע, אחוז יכול > 100% |
| `GET /api/champs/ticker` | אירועים אחרונים + `category` (analyst_success / important / team) לצבע |
| `GET/POST /api/champs/team-goal` (admin) | הגדרת יעד צוותי |

## 9. שינויים במודל DB

- **team_goals** - יעדים צוותיים (`title`, `target_value`, `current_value`, `unit`, `goal_type`, `period`, …). עבור `period=weekly` ה־target המוחזר ב־GET מחושב משבוע שעבר.
- **activity_events** - לוג לטיקר (`event_type`, `user_id`, `payload` JSON, `created_at`). הודעות עם קטגוריה לתצוגה (ירוק/כתום/כחול).
- **champ_rank_snapshots** - snapshot יומי לכל אנליסט (`user_id`, `rank`, `score`, `snapshot_date`) לחישוב מגמה (▲▼)
- **yara_rules.quality_points** - ניקוד איכות 10-50 לחוק YARA (מחושב בהעלאה/עדכון)
- **iocs:** `country_code`, `tld`, `email_domain`, `rare_find_type` - ל־Rare Find (ראשון במערכת: מדינה/TLD/דומיין אימייל)
- **IOC.user_id**, **IOC.analyst**, **YaraRule.analyst**, **UserProfile**, **UserSession** - מזוהים ופרופילים
